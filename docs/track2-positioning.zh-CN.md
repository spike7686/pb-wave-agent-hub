# Track 2 定位说明

语言版本：

- [English](./track2-positioning.md)
- 简体中文

## 提交赛道

这个仓库的定位是：

- `Track 2: Strategy Skills`

它 **不是** 一个 Track 1 风格的链上实时执行 Agent。

## 为什么适合 Track 2

Track 2 需要的是：

- 一个 CMC Skill
- 能把市场数据转换成可回测策略规格
- 不要求实时执行层

这个仓库已经以模块化方式具备这些能力：

- 把榜单快照转换成过滤后的候选池
- 使用 1h K 线和 1h OI 补充候选上下文
- 产出做空入场逻辑
- 计算止损 / 目标位结构
- 生成可回放订单
- 支持批量回测

## 面向 Track 2 的正确表达

最合适的表达方式是：

`市场数据 -> 候选排序 -> 信号生成 -> 策略规格 -> 可回放订单`

而不是：

`Binance 模拟盘机器人`

模拟盘只是一个验证表面，真正面向比赛的价值在于 strategy skill 这一层。

## 评委应该看到什么

### 技术实现

- 真正的信号生成逻辑
- 明确的止损 / 目标位构造
- 显式计入手续费 / 滑点的回放
- 可复现的 CLI 工作流

### 原创性

- 策略把榜单驱动的候选池选择和永续合约 OI 上下文结合在一起
- 它不是一个只看价格均线的通用系统

### 实用价值

- 明确的下游用户：研究员、量化开发者、信号分析师、未来执行 Agent
- 从历史回放走向未来自主执行有清晰扩展路径

### 演示清晰度

- 可以导入快照
- 可以生成历史同步计划
- 可以从这些快照生成回放订单
- 仓库结构本身就是围绕可复现性设计的

## 建议的演示口径

“这个项目是一个策略技能引擎。它把加密市场榜单快照转换成结构化、可回测的做空策略规格，里面包含明确的入场、止损、目标位和回放输出。”

## 提案时要避免的表述

不要把它包装成：

- 一个已经完成的 BSC 实盘执行 Agent
- 一个链上资产管理器
- 一个生产级经纪执行系统

这些更适合未来的 Track 1 扩展，而不是当前提交版本。
